⏺ Analysis: Why the 3 tests were originally skipped

  Root cause

  The tests were written with a real backend dependency — they required the Express API on port 4000 and DynamoDB on port 8000 to be running. Each test had a try/catch wrapper that called test.skip(true, 'Backend
  API not available') when the backend wasn't reachable. Since no backend was running during test execution, they always skipped.

  Why I designed them that way initially

  1. Took the plan too literally — The original plan spec'd e2e/helpers/api-helpers.ts with functions like seedListData() and getListData() that POST/GET directly to localhost:4000. I followed that design, making
  the sharing test POST test data to the real API before navigating.
  2. Didn't think through the page.route() option — I had already used page.route() successfully for the category-sorting tests (mocking /categories), but didn't apply the same pattern to these tests. The recipe
  and sharing endpoints are no different — they're just HTTP fetches that can be intercepted at the browser level.
  3. Over-scoped the test boundary — These tests should verify that the frontend correctly handles API responses, not that the full backend stack works. The sharing test doesn't need to actually store data in
  DynamoDB — it needs to verify that when fetchList() returns items, they render correctly.

  ⏺ Here's the situation with React 19:

  Blockers for React 19:
  - react-sortable-hoc — peer deps: react ^16.3.0 || ^17.0.0. Doesn't declare React 18 or 19 support. It also uses findDOMNode which is removed in React 19 (not just deprecated). This would break drag-and-drop
  reorder.
  - react-redux@7.1.3 — doesn't declare React 19 support. Would need upgrade to v9, but v9 requires redux@^5 (current: redux@4.0.4). That's a bigger change.
  - react-i18next@9 — peer deps say react >= 16.3.0 (open-ended), but withNamespaces() HOC is long-removed in newer versions. Unknown if it actually works with React 19 internals.

  React 18 is safer because:
  - findDOMNode is deprecated but still works (react-sortable-hoc keeps working)
  - react-redux@7.1.3 officially supports React 18
  - react-router-dom@5 works fine with React 18
  - react-i18next@9 works fine with React 18

  React 19 would require upgrading react-sortable-hoc → @dnd-kit, react-redux → v9 + redux v5, and verifying react-i18next — all at once. That's Phases 2+5+6 combined.
