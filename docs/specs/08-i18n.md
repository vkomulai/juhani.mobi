# 08 — Internationalization (i18n)

## Overview

The app supports Finnish (default) and English via `react-i18next`. Language can be switched at runtime through the settings menu, and affects both UI text and speech recognition language.

## User-Facing Behavior

### Default Language

- Finnish (`fi`) is the default and fallback language.
- Set in `i18n.init({ lng: 'fi', fallbackLng: 'fi' })`.

### Language Switching

- The settings menu shows a language toggle: displays current language code (e.g., `FI`) and the option to switch (e.g., `"Vaihda kieli (EN)"`).
- Clicking triggers `i18n.changeLanguage(newLang)`.
- All `withNamespaces()` wrapped components re-render with translated text.

### Speech Language Mapping

| UI Language | Speech API Language Code |
|------------|------------------------|
| `fi` | `fi-FI` |
| `en` (or anything else) | `en-GB` |

### Persistence

**Language does NOT persist across page reload.** The i18n configuration does not include `i18next-browser-languagedetector` or any storage plugin. Each page load starts with Finnish.

## Key Source Files

| File | Role |
|------|------|
| `src/i18n.js` | i18next initialization, `getLocaleLang()` |
| `src/translations.json` | All translation strings (Finnish and English) |
| `src/components/HamburgerMenu/Menu.jsx` | Language toggle UI, `changeLanguage()` function |
| `src/components/Button/Button.jsx` | Uses `withNamespaces()` and `t(labelKey)` |
| `src/components/InfoView/InfoView.jsx` | Uses `t()` for info texts (except hardcoded offline text) |
| `src/App.js` | Wraps app with `withNamespaces()` |

## Acceptance Criteria

1. Default language is Finnish with all Finnish text visible.
2. Switching to English updates all translated UI elements.
3. Switching back to Finnish restores Finnish text.
4. Speech recognition language follows UI language (fi → fi-FI, en → en-GB).
5. Language does NOT persist across page reload (resets to Finnish).
6. Offline mode text remains hardcoded Finnish regardless of language setting.

## Edge Cases

- The `withNamespaces()` HOC is the legacy API from `react-i18next` v9 (newer versions use `useTranslation` hook).
- `keySeparator: false` means dots in keys are literal (not nested paths).
- **Known bug**: Offline text bypass: `InfoView.jsx` returns hardcoded Finnish strings before checking i18n when `!isOnline` (see spec 06). Users who switch to English will still see Finnish text when offline, creating a language inconsistency.
